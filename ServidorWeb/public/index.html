Practica numero 2

HOLA, este servidor se comunica con el microcontrolador y con la pagina web

Por defecto estan estos valores:

  • Led
  • id : 0
  • status: 0
  • Buzzer
  • id : 1
  • status: 0
  • frecuency: 440
  • Sensor
  • id : 2
  • status: 0
  • Min distance: 50

estos valores se definen en sus respectivos controladores, en los cuales son declarados como objetos debido a que no estamos haciendo uso de una base de datos

Cada elemento tienen sus respectivas rutas, las cuales son:

  • Led
  • /led/ -> esta muestra la informacion del elemento, GET
  • /led/status -> esta cambia el valor de la variable status, no recibe parametros y devuelve el texto encendido o apagado, segun el caso, PUT
  • Buzzer
  • /buzzer/ -> esta muestra la informacion del elemento, GET
  • /buzzer/status -> esta cambia el valor de la variable status, no recibe parametros y devuelve el texto encendido o apagado, segun el caso, PUT
  • /buzzer/frecuency -> esta cambia el valor de la variable frecuency, recibe un archivo JSON con la variable frecuency y devuelve el texto exito, PUT
  • Sensor
  • /sensor/ -> esta muestra la informacion del elemento, GET
  • /sensor/status -> esta cambia el valor de la variable status, no recibe parametros y devuelve el texto encendido o apagado, segun el caso, PUT
  • /sensor/distance -> esta cambia el valor de la variable distance, recibe un archivo JSON con la variable distance y devuelve el texto exito, PUT

estos valores se definen en sus respectivos archivos de rutas en la carpeta routes
